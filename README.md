# 离线设备点检工作台

一个用于班组在本机记录设备点检的离线 Web 应用。支持计划配置、开始点检、提交结果、主管复核、关闭异常、撤销上一步、历史查询和 CSV/JSON 导出。

## 技术栈

- **前端**: React 18 + TypeScript + Vite + TailwindCSS + Zustand
- **后端**: Express + TypeScript + tsx
- **数据持久化**: 本地 JSON 文件（`data/db.json`）+ 本地图片存储（`data/evidence/`）
- **离线运行**: 完全本地部署，无需外部依赖

## 功能模块

| 模块 | 功能说明 |
|------|----------|
| 工作台 | 概览统计、最近点检单、点检计划 |
| 开始点检 | 选择计划、日期、操作员，创建点检单 |
| 执行点检 | 逐项填写正常/异常，异常上传照片证据 |
| 主管复核 | 审核异常记录，复核通过 |
| 历史查询 | 多条件筛选、导出 CSV/JSON |
| 统计分析 | 多维度数据统计、图表可视化、导出 CSV |
| 计划配置 | 管理设备、班次、检查项、点检计划 |

## 状态机流转

```
DRAFT(草稿) → SUBMITTED(已提交) → 无异常 → COMPLETED(已完成)
                    ↓
              有异常 → PENDING_REVIEW(待复核) → REVIEWED(已复核) → CLOSED(已关闭)

任意状态可通过"撤销上一步"回退到上一个状态
```

## 主流程操作步骤

### 场景 1：正常点检流程（无异常）

1. **创建点检单**
   - 点击顶部"开始点检"
   - 选择计划 "CNC-A1 日常点检"
   - 选择班次日期（默认今日）
   - 输入操作员姓名（如 "张工"）
   - 点击"创建点检单并开始"

2. **逐项检查**
   - 对每个检查项点击"正常"或"异常"
   - 全部选择"正常"
   - 点击"保存草稿"（可选）
   - 点击"提交点检单"

3. **流程自动完成**
   - 无异常时自动流转到 `COMPLETED(已完成)`
   - 主管无需复核

4. **主管关闭**
   - 切换到"主管复核"页面
   - 切换到"已复核待关闭"标签
   - 选择记录，点击"关闭异常"
   - 状态变为 `CLOSED(已关闭)`

### 场景 2：异常点检流程（带证据）

1. **创建点检单**（同场景 1 步骤 1）

2. **逐项检查**
   - 部分项选择"正常"
   - 部分项选择"异常"：
     - 展开异常项卡片
     - 选择严重程度（低/中/高）
     - 填写异常描述
     - **点击"+上传"按钮，选择至少一张照片作为证据**
     - 填写备注（可选）

3. **提交点检单**
   - 点击"提交点检单"
   - 自动流转到 `PENDING_REVIEW(待复核)`

4. **主管复核**
   - 点击"主管复核"
   - 在"待复核"标签中选择记录
   - 查看异常描述和照片证据
   - 填写处理意见（可选）
   - 点击"复核通过"
   - 状态变为 `REVIEWED(已复核)`

5. **关闭异常**
   - 切换到"已复核待关闭"标签
   - 选择记录，点击"关闭异常"
   - 状态变为 `CLOSED(已关闭)`

## 统计分析

统计分析页提供设备点检数据的多维度可视化分析，帮助管理者快速掌握设备运行状况、异常分布和趋势变化。

### 入口

在顶部导航栏点击"统计分析"进入页面。页面打开时默认显示**近30天**的数据范围。

---

### 图表口径说明

统计分析页包含四类图表，所有图表共用同一套筛选条件（日期范围 + 用户身份），切换筛选时四类图表及顶部汇总卡片**同步更新**。

| 图表名称 | 数据口径 |
|----------|----------|
| **设备维度完成率** | 按设备分组，统计筛选范围内各设备的点检单完成率。<br>公式：`完成率 = 已关闭单数 / 总开单数(排除草稿) × 100%`<br>Y 轴固定 0%~100% |
| **各班次异常率** | 按班次分组（早班/中班/夜班），统计各班级点检单的异常发生率。<br>公式：`异常率 = 含异常的单数 / 总开单数(排除草稿) × 100%` |
| **异常严重程度分布** | 饼图展示筛选范围内所有异常记录按严重程度（低/中/高）的数量占比。<br>仅展示 count > 0 的严重程度等级，全部为 0 时显示"暂无异常数据"空状态 |
| **异常趋势** | 折线图展示筛选日期范围内每一天的异常记录总数量。<br>X 轴为日期（MM-DD 格式），Y 轴为该日所有点检单中上报的异常条数之和 |

**顶部汇总卡片口径**：
- 总开单数（排除草稿）：筛选范围内所有 `status != DRAFT` 的点检单数量
- 已关闭数：筛选范围内所有 `status == CLOSED` 的点检单数量
- 异常总数：筛选范围内所有点检单中 `anomalies` 数组的长度之和
- 整体完成率：`已关闭数 / 总开单数 × 100%`，保留 1 位小数

---

### 日期筛选

- **默认值**：页面打开时默认选择"近 30 天"（从今天往前推 29 天到今天，共 30 天）
- **筛选方式**：通过两个日期选择器分别设置"开始日期"和"结束日期"（含边界）
- **生效时机**：修改任一日期后立即触发数据重新加载，loading 状态下显示加载动画
- **重置按钮**：点击"重置日期"可一键恢复到默认的近 30 天范围
- **日期格式**：接口使用 `YYYY-MM-DD` 格式字符串，按 `shiftDate` 字段过滤点检单

---

### 权限视角（主管 vs 操作员）

系统根据当前登录用户身份自动切换数据可见范围，页面右上角显示当前视角标签。

| 用户身份 | 判断规则 | 数据范围 |
|----------|----------|----------|
| **主管视角** | 用户姓名出现在任意点检单的 `supervisor` 字段中 | 查看全量数据，所有点检单均纳入统计 |
| **操作员视角** | 未在任何点检单中担任主管 | 仅统计当前用户作为 `operator` 经手的点检单 |

**注意**：
- 切换用户身份（修改"当前用户"输入框）后，筛选条件自动更新，所有图表和汇总卡片立即刷新
- 操作员视角下，若该操作员在筛选日期范围内无任何点检单，所有图表显示"该时段暂无数据"空状态
- 权限判断基于真实数据，不依赖前端输入框的标签

---

### 操作日志（持久化）

统计分析页的以下操作会自动写入本地操作日志，**重启应用后仍可查询**：

| 操作 | 触发时机 | action 标识 |
|------|----------|-------------|
| 打开页面 | 首次进入统计分析页时记录 1 次 | `VIEW` |
| 修改筛选 | 修改开始日期或结束日期时 | `FILTER` |
| 导出 CSV | 点击"导出 CSV"按钮时 | `EXPORT` |

**日志内容**：每条记录包含 `id`、`user`（操作用户）、`action`（操作类型）、`timestamp`（ISO 时间戳）、`filters`（操作时的日期范围）。

**日志位置**：
- 存储文件：`data/db.json` 中的 `statsLogs` 数组
- 查询接口：`GET /api/stats/logs`
- 写入接口：`POST /api/stats/log`

日志随 `data/db.json` 持久化，**不会因为刷新页面或重启服务器丢失**。

---

### CSV 导出说明

点击"导出 CSV"按钮可下载当前筛选条件下的统计数据 CSV 文件。

**文件特性**：
- **UTF-8 BOM**：文件头部包含 `\uFEFF` BOM 标记，Excel 打开时中文不会乱码
- **中文列头**：所有列名均为中文，便于业务人员直接阅读
- **文件命名**：`stats_{开始日期}_{结束日期}.csv`

**CSV 结构**（四个区块，用空行分隔）：

```
设备完成率统计
设备名称,总开单数(排除草稿),已关闭数,完成率(%)
CNC加工中心 A-1,5,4,80.00
...

班次异常率统计
班次名称,总开单数(排除草稿),异常单数,异常率(%)
早班,6,3,50.00
...

异常严重程度分布
严重程度,数量
低,2
中,3
高,2

近30天异常趋势
日期,异常数
2026-05-15,1
2026-05-16,0
...
```

**CSV 校验方法**：
1. 导出 CSV 后，使用 Excel 或文本编辑器打开文件
2. 检查文件第一个字节是否为 BOM（以 UTF-8 编码打开时开头不可见，以 GBK 打开时开头显示 `锘?`）
3. 逐行校验每个区块：
   - **设备完成率**：`完成率(%)` 列应等于 `已关闭数 / 总开单数 × 100`，保留 2 位小数
   - **班次异常率**：`异常率(%)` 列应等于 `异常单数 / 总开单数 × 100`，保留 2 位小数
   - **异常严重程度分布**：低+中+高的数量之和应等于页面顶部"异常总数"
   - **异常趋势**：将所有日期的异常数累加，应等于页面顶部"异常总数"
4. 将 CSV 数值与页面图表鼠标悬停时显示的 Tooltip 数值**逐项对比**，确保完全一致

### 空状态与异常处理

统计分析页针对三种异常场景提供明确的视觉反馈和操作指引：

#### 1. 加载中（loading）
- 触发：首次加载、切换筛选、点击刷新或重试按钮
- 表现：显示居中的旋转 loading 图标 + "加载中..." 文字
- 操作：所有按钮可用但刷新按钮显示 disabled 态

#### 2. 无数据（empty）
- 触发：筛选范围内无任何符合条件的点检单（排除草稿）
- 表现：
  - 顶部汇总卡片不显示
  - 四个图表区域各自独立显示空状态提示（图标 + "该时段暂无数据" + "请尝试调整日期范围或创建点检单"）
  - 每个图表的空状态保持与有数据时相同的卡片布局，视觉不跳动
- 操作：可通过"重置日期"或手动调整日期范围、创建新点检单来获得数据

#### 3. 后端异常（error）
- 触发：后端接口返回错误（5xx/4xx）或网络异常
- 表现：
  - 显示红色 Alert 图标 + "加载失败" + 具体错误信息
  - 显示红色"重试"按钮
- 操作：点击"重试"按钮重新发起请求，错误消失后正常展示数据或空状态

**状态联动**：切换筛选条件、切换用户身份时，loading / error / 数据三种状态**一起更新**，不会出现部分图表已更新、部分仍显示旧数据的情况。

## 配置导入导出

### 功能说明
支持一键导出所有配置（设备、班次、检查项、点检计划）为 JSON 文件，以及从 JSON 文件导入配置。换机器部署时无需重新手动录入。

### 导出配置
1. 进入"计划配置"页面
2. 点击右上角"导出配置"按钮
3. 自动下载 `config_export_YYYY-MM-DD.json` 文件
4. 文件包含：`devices`、`shifts`、`checkItems`、`inspectionPlans` 四类配置

### 导入配置
1. 进入"计划配置"页面
2. 点击右上角"导入配置"按钮
3. 选择导出的 JSON 配置文件
4. 系统自动进行预校验，显示：
   - 待导入的配置数量统计
   - 设备编码冲突列表（如有）
   - 引用缺失的计划列表（如有）

### 冲突处理规则

#### ✅ 设备编码重复
- 检测到系统中已存在相同编码的设备时，提供两种选择：
  - **跳过现有编码**（默认）：保留现有设备数据，跳过导入的重复项
  - **覆盖现有数据**：用导入的数据覆盖现有设备信息
- 班次和检查项通过编码自动匹配，已存在则更新，不存在则新增

#### ❌ 引用缺失（无法导入）
当计划引用的设备、班次或检查项在导入文件和现有系统中都不存在时：
- 该计划会被标记出来并跳过导入
- 红色提示框显示具体哪些计划缺少哪些引用
- 需要先修正配置文件，确保引用的配置项也包含在导入文件中

#### ❌ JSON 格式错误 / 字段缺失
- 直接报错，**一条数据都不会写入**
- 显示具体的错误信息（如第几个元素缺少哪个字段）

### 幂等性保证
重复导入同一份配置文件不会导致数据翻倍：
- 设备按 `code` 编码去重
- 班次按 `name + type + startTime + endTime` 组合去重
- 检查项按 `code` 编码去重
- 计划按 `name + deviceId + shiftId` 组合去重

### 数据持久化
- 导入的配置直接写入 `data/db.json`
- 导入成功后页面自动刷新，配置立即生效
- 重启应用后数据保持不变

## API 接口列表

### 设备管理
- `GET /api/devices` - 获取设备列表
- `POST /api/devices` - 创建设备
- `PUT /api/devices/:id` - 更新设备
- `DELETE /api/devices/:id` - 删除设备

### 班次管理
- `GET /api/shifts` - 获取班次列表
- `POST /api/shifts` - 创建班次
- `PUT /api/shifts/:id` - 更新班次
- `DELETE /api/shifts/:id` - 删除班次

### 检查项管理
- `GET /api/check-items` - 获取检查项列表
- `POST /api/check-items` - 创建检查项
- `PUT /api/check-items/:id` - 更新检查项
- `DELETE /api/check-items/:id` - 删除检查项

### 点检计划
- `GET /api/plans` - 获取计划列表
- `POST /api/plans` - 创建计划
- `PUT /api/plans/:id` - 更新计划
- `DELETE /api/plans/:id` - 删除计划

### 点检单
- `GET /api/inspections` - 获取点检单列表（支持筛选）
- `GET /api/inspections/:id` - 获取单条详情
- `POST /api/inspections` - 创建点检单
- `PUT /api/inspections/:id/results` - 更新检查结果
- `POST /api/inspections/:id/submit` - 提交点检单
- `POST /api/inspections/:id/review` - 主管复核
- `POST /api/inspections/:id/close` - 关闭异常
- `POST /api/inspections/:id/undo` - 撤销上一步

### 文件和导出
- `POST /api/evidence` - 上传异常照片
- `GET /api/evidence/:filename` - 查看照片
- `GET /api/export/csv` - 导出点检单 CSV
- `GET /api/export/json` - 导出全部数据 JSON
- `GET /api/export/order/:id/json` - 导出单条记录 JSON

### 统计分析
- `GET /api/stats/summary` - 获取统计汇总（含设备完成率、班次异常率、严重程度分布、异常趋势）
- `GET /api/stats/csv` - 导出统计数据 CSV（带 BOM）
- `POST /api/stats/log` - 记录统计操作日志
- `GET /api/stats/logs` - 获取统计操作日志列表

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| `DUPLICATE_INSPECTION` | 同一设备同一班次重复开单 |
| `ANOMALY_MISSING_EVIDENCE` | 异常缺少照片证据 |
| `INVALID_STATUS_TRANSITION` | 非法的状态流转 |
| `INVALID_OPERATION` | 无效操作（如撤销到最初状态） |
| `VALIDATION_ERROR` | 参数校验错误 |
| `NOT_FOUND` | 资源不存在 |
| `IMPORT_JSON_PARSE` | 导入文件 JSON 解析失败 |
| `IMPORT_SCHEMA_ERROR` | 导入文件字段缺失或格式错误 |
| `IMPORT_CONFLICT` | 导入存在编码冲突 |
| `IMPORT_MISSING_REF` | 导入计划引用的配置不存在 |
